# qa-report

N차 통합테스트 결과 보고서(.xlsx)를 **양식 고정·데이터만 채우는 방식**으로 결정론적 생성하는 QA용 Claude Code 플러그인. 회차마다 동일 퀄리티를 보장하고, 여러 QA가 그대로 쓸 수 있다.

- 버전: v0.1.5

## 무엇을 만드나

`/qa-report` 한 번으로 차수 결과서 `.xlsx` 1종을 생성한다. 구성:

- **Test Status** — QA 담당자 / 기간 / Build Ver.
- **상세 이슈** — BTS · 구분(신규/이월) · 결함 원인 유형 · 상세 내용 (이월 행 강조)
- **Bug Status**
  1. 이슈 수 집계 — 신규 / 이월 / 누적 발견 / 해결 완료 / 잔여 미해결 / 해결률
  2. 결함 원인 유형별 분포 — 8 고정 분류 × (건수·비율%)
  3. 차수별 추이 — 신규 발견 · 누적 발견 (회차 번다운)
- **결론/판정** — 잔여 0 → 최종 전환 / 잔여>0 → 다음 빌드 요청 (자동 제안, QA 확정)

우선순위(중요도)·차트·버그리스트 별도 양식은 다루지 않는다(count-only). 이슈 없음 후의 **최종 결과서**는 별도 양식으로 본 플러그인 범위 밖.

## 전제 조건

1. **Claude Code** (Desktop 앱의 Claude Code 통합 또는 CLI) — 로컬 파일 쓰기가 되는 환경.
2. **Atlassian(Jira) 커넥터(MCP) 연결** — 버그 데이터를 fetch한다. 미연결 시 실행 중단.
3. **Python 3 + openpyxl** — `pip install openpyxl`.
4. Jira에 **결함 원인 유형** 단일선택 커스텀필드. 없으면 해당 표는 자동 생략되고 나머지는 정상 생성.

## 사용법

```
/plugin marketplace add cmi94/qa-kit
/plugin install qa-report@qa-kit
/qa-report
```
실행하면 5개를 묻는다(강제):
1. 솔루션명  2. Jira 프로젝트 키  3. QA 일정(기간)  4. 이전 차수 결과 파일(1차면 "없음")  5. 출력 폴더

선택: 보고자(reporter, 비우면 본인) · Build Ver. · 결함 원인 유형 필드명.
나머지는 자동: 차수 N(이전 파일 메타에서 감지) · 신규/이월/해결/잔여 · 결함 원인 분포 · 차수별 추이 · 결론 제안.

### 회차 체인 (1 → 2 → 3)
- **1차**: 이전 파일 "없음" → 빈 상태에서 1차 생성.
- **2차**: 1차 결과 .xlsx 경로를 입력 → 메타 블록에서 2차 자동 감지 + 누적 이음.
- **3차**: 2차 결과 .xlsx 경로 입력 → 3차. (이하 동일)

결과 .xlsx에는 숨김 `_meta` 시트로 회차·누적·추이가 **동결 기록**되어, 다음 차수가 이를 읽어 잇는다(발부 시점 숫자 보존 = 추적성).

## 구성

```
qa-report/
  .claude-plugin/plugin.json
  commands/qa-report.md       # /qa-report 오케스트레이션 (입력→fetch→집계→렌더)
  scripts/build_report.py     # 결정론적 렌더러 (양식 RGB 고정, 메타 기록)
  scripts/read_meta.py        # 이전 차수 .xlsx 메타 읽기 (회차 감지)
  README.md
```

## 결정성

동일 입력이면 시트·스타일·문자열 XML이 전부 동일하게 나온다(생성 시각 메타만 변동). 양식이 코드에 RGB로 박혀 있어 외부 템플릿 파일에 의존하지 않는다.

## 한계

- 보고자(reporter)는 선택 입력 — 비우면 `currentUser()`(실행 본인 등록분), 지정 시 그 보고자(들) 등록분 집계(타 QA·팀 데이터 추출 가능).
- 결함 원인 유형 커스텀필드 id는 Jira 인스턴스마다 다름 — 필드명으로 매칭, 부재 시 표 생략.
