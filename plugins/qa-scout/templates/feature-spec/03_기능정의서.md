## 16번 인풋 출처 / 17번 비고 — evidence·deep screen mapping hint

본 플러그인 spec(`../../../docs/qa-scout/spec.md` v0.2.8 footer 참조)에 따라 deep screen target 행은 다음 양식으로 출처·비고를 채운다. 단순 PRD §x.x 1줄만으로는 깊은 화면 누락을 잡을 수 없다.

### 16번 인풋 출처 — 다중 evidence 인용

`(<문서 근거>; <화면 근거>; <코드 근거>; BR-<도메인>-NN)` 4종 중 가능한 만큼 ;-구분으로 나열.

| evidence 종류 | 예시 표기 |
|---|---|
| 문서 (PRD/UC/매뉴얼) | `PRD §3.4`, `UC §<도메인>-UC-<NNNN>`, `manual §5.2` |
| 화면 capture | `capture:<screen-slug>.yaml#buttons[0]`, `capture:<panel-slug>.yaml#tables[1]` |
| design 근거 | `design:<설계 문서명> §3.1` |
| 코드 근거 | `code:<src/경로/파일>:<라인>` (확정 단정 X — evidence로만) |
| 권한 매트릭스 | `permission-matrix §<UC 코드>` |
| 상태 전이 | `state-transition §<도메인 엔티티>`, `UC §상태별 허용 행위` |
| BR 코드 | `BR-<도메인>-NN` |

확정 1건만 있고 나머지가 부족하면 `[자료 부족 — <차원>]`을 함께 표기. 예: `(PRD §3.4; [자료 부족 — 화면 capture 없음])`.

### 17번 비고 — deep screen / behavior marker

해당 행이 deep screen target에 속하거나 동적 동작·변수·상태 분기를 포함하면 아래 marker 중 해당하는 것을 비고에 명시한다. reviewer가 enum 판정에 활용한다.

| marker | 의미 | reviewer 1차 enum |
|---|---|---|
| `[상세 화면 구조 부족]` | 화면 capture가 inputs:[]이거나 패널/모달 미관찰 | SCREEN-MISSING |
| `[동적 UI 확인 필요]` | 클릭·입력 후 UI 변화 또는 lifecycle 미관찰 | BEHAVIOR-MISSING |
| `[변수 동작 자료 부족]` | 변수 marker·atomic·PDF 치환 lifecycle 미확정 | VARIABLE-MISMATCH |
| `[상태별 UI 확인 필요]` | Draft/Approved/Pending 등 상태별 분기 미확정 | STATE-MISMATCH |
| `[권한별 UI 확인 필요]` | role별 노출·동작 차이 미확정 | PERMISSION-MISMATCH |
| `[위험 액션 미검증]` | 저장/삭제/승인/제출/신규 버전 생성 등 자동 클릭 금지 | NOT-TESTED-RISKY-ACTION |
| `[문서-화면 충돌]` | 문서·화면 한쪽만 있거나 둘이 충돌 | SCREEN-MISSING / SPEC-MISSING / FAIL |

기존 마커(`[자료 부족]`, `[화면 확인 필요]`, `[문서 근거 부족]`, `[사용자 시나리오 확인 필요]`)는 그대로 유지한다.

### deep screen target 행 예시

| 9 (상세 정책) | 16 (인풋 출처) | 17 (비고) |
|---|---|---|
| `<도메인 엔티티>` Draft 상태에서만 `<자식 엔티티>` 순서를 드래그&드롭으로 변경 가능 (UC-NNNN 대안흐름) | `(UC §<도메인>-UC-<NNNN>; capture:<screen-slug>.yaml#tables[1]; state-transition §<도메인 엔티티>; [자료 부족 — 드래그 lifecycle 화면 미관찰])` | `[동적 UI 확인 필요]` `[상태별 UI 확인 필요]` — deep_screen_targets[].id=`SCR-<PROJECT>-<screen-slug>` |
| 변수 marker는 `img.<class>`로 렌더링되며 Backspace 1회로 atomic 삭제 | `(design:<설계 문서명> §4.1; UC §없음 — 문서 근거 부족)` | `[변수 동작 자료 부족]` — gap=variable-behavior-gap |
