# 사용자스토리 — {프로젝트명}

## 9컬럼 정의

| # | 컬럼 | 비고 |
|---|---|---|
| 1 | Story ID | `US-MYAPP-NNN` |
| 2 | 에픽 (Epic) | 묶음 단위 (예: 사용자 온보딩) |
| 3 | Sprint | S1·S2·S3 |
| 4 | 사용자 스토리 | "As a ___, I want ___, so that ___" |
| 5 | 인수 기준 (G/W/T) | "Given ___ / When ___ / Then ___" |
| 6 | 관련 기능 ID | `FR-MYAPP-NNN` 다중 |
| 7 | 포인트 (SP) | 1·2·3·5·8 (Fibonacci) |
| 8 | 우선순위 | M / S / C / W |
| 9 | 상태 | Backlog·In Progress·Done |

## 채움 양식 (예시)

| 1 | 2 | 3 | 4 | 5 | 6 | 7 | 8 | 9 |
|---|---|---|---|---|---|---|---|---|
| US-MYAPP-001 | 사용자 온보딩 | S1 | "As a 사이트 관리자, I want 신규 입사자 계정을 생성하기, so that 작업자가 MYAPP 시스템에 로그인해 공정을 수행할 수 있다" | "Given 관리자 권한(USER_WRITE) 보유 / When [사용자 등록] 버튼 클릭 후 ID/이름/부서 입력 / Then 계정이 ACTIVE 상태로 생성되고 passwordChangedAt=null로 최초 로그인 비밀번호 변경이 강제되며 Audit Trail에 자동 기록된다" | FR-MYAPP-001 | 5 | M | Backlog |

## 채움 가이드 (단계 9 — scout 본 절차)

- PRD·유스케이스·Process Flow에서 도출
- "As a / I want / so that" 형식 엄수
- 인수 기준은 Given/When/Then 명확 분리
- 관련 FR ID 다중 가능 (한 스토리가 여러 기능 매핑)
- 단계 17~20에서 사람 검수 시 우선순위·SP 검토
